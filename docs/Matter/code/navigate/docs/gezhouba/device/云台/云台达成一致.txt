1. 云台在长时间不掉电的情况下运转，存在精度和不一致的情况:
	1/ 物理机械精度损失累计
	2/ 物理精度和软件回显精度累计
  导致的问题和风险:
    误差累积一般是5度左右 -- 长时间累计根据雷达的测试长度在500-800米偏差会很大
	 
  解决方案：
    1. 补偿伺服器 可以实现长时间运行 --- 成本和工作量考虑暂时搁置
    2. 每隔4-5小时断电重启自检 ------ 可以解决1/ 2/ 需要自己测试 

  当前选择 解决方案：2.
  遇到的问题: 
	1- 每次重启自检需要1-2分钟 考虑负重情况下可能会3分钟，需要结合运营情况来设计
	2- 每次重启需要重新用软件设置角度参数，测试回显的角度是否和标准一致
